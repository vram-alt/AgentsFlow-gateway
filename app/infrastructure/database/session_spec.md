# Спецификация: Управление подключением к БД (session.py)

> **Файл реализации:** `session.py`  
> **Слой:** Infrastructure (внешний мир — I/O)  
> **Ответственность:** Создание и конфигурация асинхронного движка SQLAlchemy, фабрики сессий, инициализация БД

---

## 1. Общие правила

- Все операции с БД — **асинхронные** (через AsyncSession).
- Драйвер: aiosqlite для SQLite, переключаемый на PostgreSQL через DATABASE_URL.
- Обязательное включение WAL-режима при инициализации SQLite.
- ORM-модели SQLAlchemy **не экспортируются** за пределы слоя infrastructure.

---

## 2. Движок (Engine)

- Создаётся через асинхронную фабрику движков SQLAlchemy с параметром DATABASE_URL.
- Параметры: echo=False (в production), pool_pre_ping=True.
- **Параметры пула соединений (обязательные):** размер пула — 5 соединений, максимальное переполнение — 10 дополнительных соединений сверх размера пула. Эти значения рекомендуется вынести в конфигурацию приложения для возможности настройки под нагрузку. Явное ограничение пула предотвращает исчерпание file descriptors и лимитов БД при пиковой нагрузке.

---

## 3. WAL-режим для SQLite

- При каждом новом соединении к SQLite-движку система должна автоматически выполнить две SQL-команды:
  - Включить журнал в режиме WAL (journal_mode=WAL).
  - Установить таймаут ожидания блокировки в 5000 мс (busy_timeout=5000).
- Механизм: подписка на событие установления соединения на уровне синхронного движка.

---

## 4. Фабрика сессий (SessionLocal)

- Асинхронная фабрика сессий, привязанная к движку.
- Параметры: использовать асинхронный класс сессии, отключить автоматическое истечение объектов после коммита (expire_on_commit=False).

---

## 5. Функция: get_db_session

### Назначение

Асинхронный генератор для FastAPI Depends. Возвращает AsyncSession.

### Пошаговая логика

1. Открывает сессию через фабрику.
2. Yield-ит сессию вызывающему коду.
3. В finally-блоке закрывает сессию.
4. При исключении — выполняет rollback перед закрытием.

---

## 6. Функция: init_db

### Назначение

Создаёт все таблицы через механизм метаданных декларативного базового класса.

### Пошаговая логика

1. Открывает соединение через движок.
2. Выполняет создание всех таблиц (create_all).
3. Вызывается один раз при старте приложения (lifespan event).

---

## 7. Обработка ошибок

| Ситуация                        | Действие                                    |
|---------------------------------|---------------------------------------------|
| Ошибка SQLAlchemy при подключении | Логирование + повторная попытка           |
| Невалидный DATABASE_URL         | ValueError при старте приложения            |
| Ошибка подключения к БД         | OperationalError → логирование + re-raise   |
