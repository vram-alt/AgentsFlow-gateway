# Спецификация: app/api/schemas/tester.py

## Назначение

Pydantic V2 схемы для эндпоинтов модуля «Testing Console». Реализует файл `app/api/schemas/tester.py`.

---

## §1. Схема: TesterProxyRequest

Тело запроса для POST /api/tester/proxy.

| Поле | Тип | Обязательное | По умолчанию | Валидация |
|---|---|---|---|---|
| `provider_name` | str | Да | — | Минимальная длина: 1 символ. |
| `method` | str | Нет | "POST" | Допустимые значения (регистронезависимо): "GET", "POST", "PUT", "DELETE". Приводить к верхнему регистру. |
| `path` | str | Нет | "/chat/completions" | См. §1.1 (расширенная валидация). |
| `body` | dict или None | Нет | None | См. §1.2 (ограничение размера). |
| `headers` | dict или None | Нет | None | См. §1.3 (ограничение количества и длины). |

### §1.1 Валидация поля `path`

Валидатор поля `path` ДОЛЖЕН выполнять следующие проверки:
1. URL-декодировать значение `path` перед проверкой (для предотвращения обхода через percent-encoding, например `%2e%2e` вместо `..`).
2. Проверить, что декодированный `path` не содержит подстроку `://` (запрет абсолютных URL).
3. Проверить, что декодированный `path` не содержит подстроку `..` (запрет path traversal).
4. При нарушении любого условия — отклонить с ошибкой валидации.

### §1.2 Валидация поля `body` (Ограничение размера)

Поле `body` типа `dict | None` ДОЛЖНО проходить дополнительную валидацию через model_validator:
1. Если `body` не None — сериализовать его в JSON-строку и проверить, что длина строки не превышает 1 048 576 байт (1 МБ).
2. При превышении — отклонить с ошибкой валидации "Request body too large (max 1MB)".
3. Это предотвращает отправку JSON-тела размером в сотни мегабайт и защищает от глубоко вложенных структур, вызывающих `RecursionError`.

### §1.3 Валидация поля `headers` (Ограничение количества и длины)

Поле `headers` типа `dict | None` ДОЛЖНО проходить дополнительную валидацию:
1. Максимальное количество заголовков — 20. При превышении — отклонить с ошибкой "Too many headers (max 20)".
2. Максимальная длина ключа заголовка — 128 символов. При превышении — отклонить с ошибкой "Header name too long (max 128 chars)".
3. Максимальная длина значения заголовка — 4096 символов. При превышении — отклонить с ошибкой "Header value too long (max 4096 chars)".
4. Это предотвращает атаку через передачу тысяч заголовков с произвольно длинными значениями.

## §2. Схема: TesterProxyResponse

Ответ для POST /api/tester/proxy (успех).

| Поле | Тип | Описание |
|---|---|---|
| `status_code` | int | HTTP-статус ответа от провайдера. |
| `headers` | dict со строковыми ключами и значениями | Отфильтрованные заголовки ответа. |
| `body` | Any (dict, list или str) | Тело ответа от провайдера (JSON или сырой текст). |
| `latency_ms` | float | Время выполнения запроса в миллисекундах. |

## §3. Схема: TesterErrorResponse

Ответ об ошибке для эндпоинтов тестера.

| Поле | Тип | Описание |
|---|---|---|
| `trace_id` | str | UUID v4 для трассировки. |
| `error_code` | str | Код ошибки (PROVIDER_NOT_FOUND, PROXY_TIMEOUT, PROXY_CONNECTION_ERROR, VALIDATION_ERROR, INTERNAL_ERROR, RESPONSE_TOO_LARGE). |
| `message` | str | Человекочитаемое описание ошибки. |
