# Спецификация: Роутер UI-страниц (ui_pages.py)

> **Файл реализации:** `ui_pages.py`  
> **Слой:** API / Delivery  
> **Ответственность:** HTTP-обработчики для отдачи HTML-страниц административного интерфейса

---

## 1. Общие правила

- Роуты — тонкие обёртки: отдают HTML-страницы через шаблонизатор.
- Бизнес-логика **запрещена** в роутах.
- Все зависимости получаются через FastAPI Depends.
- UI-роуты не имеют префикса /api/.

---

## 2. Роутер

**Префикс:** `/` (без `/api/`)  
**Теги:** "UI"  
**Защита:** Все UI-страницы — HTTP Basic Auth.

---

## 3. Эндпоинт: GET /

### Назначение

Главная страница (дашборд).

### Ответ

HTML через Jinja2 TemplateResponse.

---

## 4. Эндпоинт: GET /chat

### Назначение

Страница чата (отправка промптов).

---

## 5. Эндпоинт: GET /policies

### Назначение

Страница управления политиками.

---

## 6. Эндпоинт: GET /logs

### Назначение

Страница журнала событий.

---

## 7. Эндпоинт: GET /providers

### Назначение

Страница управления провайдерами.

---

## 8. Обработка ошибок

| HTTP-статус | Когда                                    |
|-------------|------------------------------------------|
| 200         | Успешная отдача HTML-страницы            |
| 401         | Невалидный токен / не авторизован        |
| 500         | Внутренняя ошибка сервера                |
