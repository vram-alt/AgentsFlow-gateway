# Спецификация: Доменная сущность LogEntry (log_entry.py)

> **Файл реализации:** `log_entry.py`  
> **Слой:** Domain (чистое ядро, 0 внешних зависимостей кроме Pydantic)  
> **Ответственность:** Описание бизнес-сущности «Запись журнала событий» — единая полиморфная запись аудита

---

## 1. Общие правила

- Модель наследуется от BaseModel (Pydantic).
- Используется Pydantic V2 (конфигурация через ConfigDict).
- Поля с датами имеют тип datetime. Значение по умолчанию задаётся через механизм фабрики (default_factory), которая вызывает получение текущего UTC-времени **при каждом создании экземпляра**, а не один раз при импорте модуля. Использовать timezone-aware вариант получения текущего времени (с явным указанием часового пояса UTC), т.к. вариант без timezone deprecated начиная с Python 3.12.
- Модель **не содержит** логики работы с БД — это чистый доменный объект.

---

## 2. Перечисление EventType

Строковое перечисление (наследует от строки и Enum) с тремя допустимыми значениями:

| Значение константы     | Строковое представление    | Описание                              |
|------------------------|----------------------------|---------------------------------------|
| CHAT_REQUEST           | "chat_request"             | Событие отправки промпта и ответа LLM |
| GUARDRAIL_INCIDENT     | "guardrail_incident"       | Событие срабатывания Guardrail        |
| SYSTEM_ERROR           | "system_error"             | Системная ошибка                      |

---

## 3. Сущность: LogEntry

### Назначение

Единая полиморфная запись аудита. Хранит события чата, инциденты Guardrail и системные ошибки.

### Поля

| Поле           | Тип              | Обязательное | По умолчанию     | Описание                                      |
|----------------|------------------|--------------|------------------|-----------------------------------------------|
| `id`           | целое или None   | Нет          | None             | Первичный ключ (назначается БД)               |
| `trace_id`     | строка           | Да           | —                | UUID сквозного идентификатора (индексируемый)  |
| `event_type`   | EventType        | Да           | —                | Тип события (Enum)                            |
| `payload`      | словарь          | Да           | —                | Полиморфное тело события (JSON)               |
| `created_at`   | datetime         | Нет          | текущее UTC-время (через фабрику) | Дата создания записи              |

### Валидация

- `trace_id`: формат UUID v4 (36 символов с дефисами).
- `event_type`: строго одно из значений Enum EventType.
- `payload`: не пустой словарь.

---

## 4. Обработка ошибок

Модель при невалидных данных выбрасывает стандартный pydantic.ValidationError.
Никакой кастомной обработки ошибок на уровне сущности не предусмотрено — это ответственность вышестоящих слоёв (services, api).
