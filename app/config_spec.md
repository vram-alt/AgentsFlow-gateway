# Спецификация: Конфигурация приложения (config.py)

> **Файл реализации:** `config.py`  
> **Слой:** Корневой  
> **Ответственность:** Централизованное чтение переменных окружения через Pydantic Settings

---

## 1. Общие правила

- Конфигурация реализуется через механизм Pydantic Settings (наследование от BaseSettings).
- Все секреты читаются из `.env` файла (путь к файлу задаётся в конфигурации модели).
- Класс `Settings` — синглтон (через механизм мемоизации, один экземпляр на процесс).
- Никакие секреты **не хардкодятся** в исходном коде.

---

## 2. Класс: Settings

### Поля

| Поле                    | Тип    | Обязательное | По умолчанию                              | Env-переменная          |
|-------------------------|--------|--------------|-------------------------------------------|-------------------------|
| `database_url`          | `str`  | Нет          | `"sqlite+aiosqlite:///./gateway.db"`      | `DATABASE_URL`          |
| `admin_username`        | `str`  | Да           | —                                         | `ADMIN_USERNAME`        |
| `admin_password`        | `str`  | Да           | —                                         | `ADMIN_PASSWORD`        |
| `webhook_secret`        | `str`  | Да           | —                                         | `WEBHOOK_SECRET`        |
| `encryption_key`        | `str`  | Да           | —                                         | `ENCRYPTION_KEY`        |
| `external_http_timeout` | `int`  | Нет          | `30`                                      | `EXTERNAL_HTTP_TIMEOUT` |

### Валидация

- `database_url`: непустая строка.
- `admin_username`: обязательное поле без значения по умолчанию. Валидатор должен запрещать тривиальные предсказуемые значения (такие как "admin", "root", "administrator") — при обнаружении выбрасывать ошибку валидации.
- `admin_password`: минимум 12 символов. Валидатор должен проверять наличие хотя бы одной цифры и хотя бы одного спецсимвола. При несоответствии — выбрасывать ошибку валидации с описанием требований к сложности пароля.
- `webhook_secret`: минимум 16 символов.
- `encryption_key`: обязательное поле. Валидатор должен проверять, что значение является корректным Fernet-ключом (строка длиной ровно 44 символа в кодировке base64). При несоответствии — выбрасывать ошибку валидации с описанием формата ключа. Ключ используется для симметричного шифрования API-ключей провайдеров в БД.
- `external_http_timeout`: положительное целое число, диапазон [5, 120].

### Требования к файлу `.env.example`

- Файл `.env.example` обязан содержать **валидные примеры**, проходящие все валидаторы выше. Запрещено указывать тривиальные значения (например, "admin" для имени пользователя или пароль без цифр и спецсимволов), так как приложение не запустится с невалидными значениями и введёт разработчика в заблуждение при первом запуске.

---

## 3. Функция: get_settings

### Назначение

Возвращает кэшированный (синглтон) экземпляр `Settings`. Кэширование реализуется через механизм мемоизации (один экземпляр на весь процесс). Используется как зависимость (Depends) в FastAPI-роутах.

### Входные параметры

Нет.

### Возвращаемое значение

Экземпляр `Settings`.

---

## 4. Обработка ошибок

| Ситуация                        | Действие                                          |
|---------------------------------|---------------------------------------------------|
| `.env` файл не найден           | Использовать значения по умолчанию + warning      |
| Обязательное поле не задано     | `ValidationError` при старте → приложение не запустится |
| Невалидное значение             | `ValidationError` при старте → приложение не запустится |
